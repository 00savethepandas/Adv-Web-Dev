JavaScript Objects and Arrays

Arrays

nFriendly Web Development

Arrays are created using brackets []. var arr = []; (shortcut for new array).

arr[0] = "cat"; // adds to the array
arr[1] = "mouse"; // adds to the array
arr[2] = "elephant"; // adds to the array
arr["human"] = "Bob"; // doesn't add to the array.
// setting a string parameter adds to the underlying object?

arr.length; // returns 3

Length is always 1 higher than the highest index, even if there are fewer items in the array.


var arr = [];
arr.length; // returns 0

arr[100] = "one is the lonliest number.";

arr.length; // returns 101.

They are very similar to setting object properties except that objects use a string while arrays use a number.

Arrays length property tells how many items are in the array and is automatically updated when something is added or removed from the array.

You can manually set the length property, var myArray = [100];

Some built-in array functions:

  • .sort()
  • .slice()
  • .push()
  • .pop()

These functions set arrays apart from array-like objects.

Array-Like Objects

Array-like objects look like arrays, have numbered elements, and a length property. They don't have any of the array functions and for-in loops don't work.

Common Array-like objects:

  • arguements variable
  • html node sets returned by document.getElementByTagName()
  • document.forms
  • Every other DOM method and property that gives a list of items
document.forms.length; // returns 1;
document.forms[0]; // returns a form element
document.forms(", "); // returns a type error. This is not an array. typeof document.forms; // returns "object"

You can send any number of parameters you want to a javascript function. They are stored in array-like object named arguements.
